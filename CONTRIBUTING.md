# Contributing to Agent Mag

Thanks for your interest in contributing to Agent Mag's open-source tools. We welcome contributions from the AI agent builder community.

## Getting Started

1. Fork the repo you want to contribute to
2. Create a feature branch (`git checkout -b feat/your-feature`)
3. Make your changes
4. Run any existing tests (`npm test` or `pnpm test`)
5. Commit with a clear message
6. Open a pull request

## Guidelines

- **One PR, one concern.** Keep pull requests focused on a single change.
- **Test your changes.** If the repo has tests, make sure they pass. If you're adding a feature, add tests.
- **Document what matters.** Update the README if your change affects how people use the tool.
- **Be respectful.** See our [Code of Conduct](CODE_OF_CONDUCT.md).

## Reporting Bugs

Open an issue in the relevant repo with:

- What you expected to happen
- What actually happened
- Steps to reproduce
- Your environment (OS, Node version, etc.)

## Suggesting Tools

Have an idea for a new tool Agent Mag should build? [Start a discussion](https://github.com/orgs/Agent-mag/discussions) with:

- The problem it solves
- Who it helps
- Any existing alternatives and why they fall short
