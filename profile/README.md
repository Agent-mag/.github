## ⚡ Open Skills Registry — `npx agentmag add`

Our headline open-source project. An open registry of installable agent skills — config bundles (prompts, tools, manifests) you can add to any agent framework with a single command:

```bash
npx agentmag add <skill-name>
```

### How It Works

1. **Browse** skills at [theagentmag.com/skills](https://theagentmag.com/skills) — 15+ skills across 6 categories
2. **Install** with `npx agentmag add <skill>` — drops a config bundle into your project
3. **Audit** everything — it's prompts and config, not executable code
4. **Customize** — tweak the prompt, adjust tool schemas, make it yours

### Skill Categories

| Category | What's Inside |
|----------|--------------|
| 🔍 **Browsing & Search** | Web search, site crawling, content extraction |
| 💻 **Code & Compute** | Code execution, sandboxed environments, REPL access |
| 📁 **Data & Files** | File parsing, data transformation, document processing |
| 💬 **Communication** | Email, messaging, notification integrations |
| 🧠 **Memory & Knowledge** | RAG, vector stores, knowledge graph connectors |
| 🎨 **Multimodal** | Vision, audio, image generation, OCR |

## 🤝 Why We Open-Source

We're not open-sourcing because it's trendy. We're doing it because we think it's the **only honest way to serve a builder community**.

- **Builders trust tools they can read.** Every line of code in this org is auditable. No black boxes.
- **The best tools come from real problems.** Every repo here started as something we needed ourselves while building agent systems.
- **Community makes tools better.** A prompt template that works for one use case gets battle-tested across dozens when it's open.
- **We want to earn attention, not buy it.** If our free tools are genuinely useful, builders will check out the rest of the platform. That's the deal.

**Our commitment:** everything in this org stays free, MIT-licensed, and maintained. No bait-and-switch. No open-core gotchas.

## 👋 Contributing

We welcome contributions from anyone building with AI agents. The platform itself is proprietary, but the ecosystem is fully open.

### Where to Contribute

| What You Want to Do | Where |
|---------------------|-------|
| **Submit a new agent skill** | PR to [`Agent-mag/skills`](https://github.com/Agent-mag/skills) — add a config bundle (prompt, tools, manifest) |
| **Submit a new tool** | PR to [`Agent-mag/tools`](https://github.com/Agent-mag/tools) — add a standalone utility, CLI, or library |
| **Propose a feature** | Issue on [`Agent-mag/roadmap`](https://github.com/Agent-mag/roadmap) — describe what and why |
| **Propose a large feature (RFC)** | PR to [`Agent-mag/roadmap`](https://github.com/Agent-mag/roadmap) with a proposal doc |
| **Report a bug** | Open an issue in the relevant repo |
| **Write an article** | Pitch at [editorial@theagentmag.com](mailto:editorial@theagentmag.com) |

### How It Works

1. You contribute to the **public repos** (skills, tools, roadmap)
2. Automated checks validate your PR (schema, security, quality)
3. We review and merge
4. The Agent Mag platform **syncs from these repos** — your contribution goes live on [theagentmag.com](https://theagentmag.com)

**You never need access to the private codebase.** The public repos are the interface.

### Principles

1. **Solve a real problem** — address friction that builders actually face in production
2. **Keep it focused** — one tool, one job, done well. No Swiss Army knives.
3. **Document like a builder** — install to working in under 5 minutes
4. **Test what matters** — confidence that the tool works as advertised
