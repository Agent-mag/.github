## Why We Open-Source

Agent Mag is a platform for AI agent builders — articles, tools, skills, jobs, events, and more. But we realized that the best way to earn the trust of builders is to **actually build useful things and give them away**.

Every tool in this org exists because we hit a real friction point while building agent systems ourselves. Instead of keeping solutions internal, we publish them here — free, MIT-licensed, and maintained.

**Our bet:** if we give builders the best free tools, they'll stick around for everything else.

---

## Open Skills Registry

An open registry of installable agent skills — config bundles (prompts, tools, manifests) you can drop into any agent framework:

```bash
npx agentmag add <skill-name>
```

Each skill is a config bundle — **not executable code** — so you can audit everything before it touches your system. Browse the full registry at [theagentmag.com/skills](https://theagentmag.com/skills).

## Contributing

We welcome contributions. Each repo has its own guide, but the principles are universal:

1. **Solve a real problem** — address a friction point builders actually face
2. **Keep it focused** — one tool, one job, done well
3. **Document clearly** — install to working in under 5 minutes

Have an idea for a tool? [Open a discussion](https://github.com/orgs/Agent-mag/discussions) or reach out at [hello@theagentmag.com](mailto:hello@theagentmag.com).
